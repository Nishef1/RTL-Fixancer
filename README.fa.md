## RTL Fixancer

RTL Fixancer یک افزونه Chrome برای بهبود تایپوگرافی و جهت‌دهی متن‌های راست‌به‌چپ در وب است. افزونه متن‌های RTL را در سطح هر عنصر تشخیص می‌دهد، جهت و تراز را اصلاح می‌کند، فونت خواناتر اعمال می‌کند و در صفحات مدرن و رابط‌های چت هوش مصنوعی با متن‌های ترکیبی فارسی/عربی/عبری/انگلیسی بهتر کار می‌کند.

### ویژگی‌های برجسته

- **پشتیبانی چندزبانه RTL**: فارسی، عربی و عبری با ماژول‌های جدا پشتیبانی می‌شوند.
- **هسته مشترک تشخیص زبان**: فایل `rtl-common.js` helperهای مشترک تشخیص و اعمال تایپوگرافی RTL را نگه می‌دارد تا منطق تکراری بین زبان‌ها ساخته نشود.
- **رابط انگلیسی به‌صورت پیش‌فرض**: پاپ‌آپ افزونه به‌صورت پیش‌فرض انگلیسی است و متن‌های UI در `ui-i18n.js` متمرکز شده‌اند.
- **قابل استفاده روی همه سایت‌ها**: هر سایت عادی `http`/`https` را می‌توانید از پاپ‌آپ یا منوی راست‌کلیک فعال کنید.
- **بهینه برای چت‌های هوش مصنوعی**: برای ChatGPT، Perplexity، Google AI Studio، Gemini، DeepSeek و رابط‌های مشابه بهتر عمل می‌کند.
- **سازگار با متن ترکیبی**: برای پاراگراف‌هایی که هم انگلیسی و هم متن RTL دارند، بدون دست‌کاری بلوک‌های کد و inputها، direction و فونت را بهتر اعمال می‌کند.
- **فونت مناسب**: برای فارسی از Vazir/Shabnam و برای عربی/عبری از fallbackهای سیستمی مناسب استفاده می‌شود.
- **خروجی PDF**: از پاپ‌آپ یا context menu می‌توانید صفحه/چت را وارد print dialog کنید و به PDF ذخیره کنید.
- **سخت‌سازی MV3**: افزونه Manifest V3 است، از `host_permissions` گسترده استفاده نمی‌کند و عملیات scripting را به URLهای قابل پشتیبانی محدود می‌کند.

### زبان‌های RTL پشتیبانی‌شده

| زبان | فایل | توضیح |
| --- | --- | --- |
| فارسی | `rtl-common.js` | تشخیص حروف فارسی و استفاده از Vazir/Shabnam. |
| عربی | `languages/arabic.js` | تشخیص بازه‌های عربی و font stack مناسب عربی. |
| عبری | `languages/hebrew.js` | تشخیص بازه‌های عبری و font stack مناسب عبری. |

### روی چه سایت‌هایی کار می‌کند؟

RTL Fixancer روی هر سایت عادی که بازدید می‌کنید قابل فعال‌سازی است. کافی است سایت فعلی را از پاپ‌آپ یا منوی راست‌کلیک فعال کنید. کاربرد اصلی آن مخصوصاً در رابط‌های چت هوش مصنوعی است، جایی که فارسی، عربی، عبری و انگلیسی ممکن است در یک پاسخ ترکیب شوند.

پشتیبانی بهینه برای این سایت‌ها در نظر گرفته شده است:

- `chat.openai.com` / `chatgpt.com`
- `perplexity.ai`
- `aistudio.google.com` / `makersuite.google.com`
- `gemini.google.com`
- `deepseek.com`
- هر سایت دیگری با متن RTL

به‌زودی.

### نصب دستی برای توسعه‌دهنده

1. این ریپو را دانلود یا clone کنید.
2. Chrome را باز کنید و وارد `chrome://extensions` شوید.
3. گزینه **Developer mode** را فعال کنید.
4. روی **Load unpacked** کلیک کنید و پوشه ریشه ریپو، همان پوشه‌ای که `manifest.json` دارد، را انتخاب کنید.
5. افزونه را pin کنید، یک سایت را باز کنید و همان سایت را از پاپ‌آپ فعال کنید.

## استفاده

- سایت فعلی را از پاپ‌آپ فعال/غیرفعال کنید.
- فونت را انتخاب کنید: `Vazir`، `Shabnam` یا پیش‌فرض مرورگر.
- حساسیت تشخیص را تنظیم کنید.
- روی **PDF** کلیک کنید تا print dialog مرورگر باز شود و بتوانید صفحه/چت را PDF کنید.
- از context menu برای کارهای سریع مثل toggle سایت، re-apply و export PDF استفاده کنید.

## مجوزها

- `storage`: ذخیره تنظیمات مثل فونت، اندازه، حالت تشخیص، زبان UI و سایت‌های فعال.
- `activeTab` و `scripting`: اجرای content script برای actionهای کاربر.
- `tabs` و `contextMenus`: مدیریت آیکون افزونه و منوی راست‌کلیک.

Manifest دیگر `host_permissions` گسترده مثل `<all_urls>` درخواست نمی‌کند. content script روی صفحات match شده اجرا می‌شود، اما رفتار افزونه بر اساس سایت‌هایی است که خودتان فعال می‌کنید.

## حریم خصوصی

- بدون analytics.
- بدون سرور خارجی.
- متن کاربر به جایی ارسال نمی‌شود.
- همه پردازش‌ها محلی داخل مرورگر انجام می‌شوند.
- فونت‌های بسته‌بندی‌شده از خود افزونه لود می‌شوند، نه از شبکه.
- خروجی PDF از print dialog مرورگر/سیستم استفاده می‌کند.

## معماری

- **helperهای مشترک RTL**: `rtl-common.js`
- **ماژول‌های زبان**: `languages/arabic.js` و `languages/hebrew.js`
- **موتور اصلی content**: `content.js`
- **patchهای سازگاری**: `content-patch.js` و `content-rtl-upgrade.js`
- **سرویس‌ورکر پس‌زمینه**: `background.js`
- **رابط پاپ‌آپ**: `popup.html`، `popup.js` و `popup-patch.js`
- **ترجمه‌های UI**: `ui-i18n.js`
- **کمک‌کننده PDF**: `lib/print-helper.js`

## نکات توسعه

- افزونه Manifest V3 است.
- content script در `document_start` اجرا می‌شود.
- برای محتوای داینامیک از `MutationObserver` و `requestIdleCallback` استفاده می‌شود.
- ناحیه‌های code/pre/input/textarea نادیده گرفته می‌شوند تا کدها و فیلدهای ورودی خراب نشوند.
- فعال‌سازی سایت‌ها subdomain-aware است.
- زبان پیش‌فرض پاپ‌آپ از طریق `ui-i18n.js` انگلیسی است.
